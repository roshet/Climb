# Champ Select Coaching Sentence Design

## Goal

Add the Gemini-generated coaching sentence to the existing focus card in the champ-select overlay, giving the player actionable advice right before the game starts.

## Context

The champ-select overlay (`src/champ-select/App.tsx`) already has a `FocusCard` component that shows the player's champion-specific top issue with stats (games seen, avg gold lost). What it lacks is the coaching sentence generated by Gemini â€” the one line of concrete advice that makes the Chat tab focus card useful.

## Architecture

Single file change: `src/champ-select/App.tsx`. No backend changes.

**Data source:** `/focus` endpoint (already exists). Returns `coaching_sentence: string` alongside other fields. Returns null if no focus card exists.

**Fetch strategy:** One-time `useEffect` on mount, not wired into the 2-second `/champ-select` poll loop. The coaching sentence doesn't change mid-champ-select and doesn't need to be refreshed.

**Failure handling:** If `/focus` returns null or the fetch fails, `coachingSentence` stays null and the existing `FocusCard` renders unchanged.

## Changes

### New state

```tsx
const [coachingSentence, setCoachingSentence] = useState<string | null>(null)
```

### New useEffect

```tsx
useEffect(() => {
  fetch(`http://localhost:${port}/focus`)
    .then(r => r.ok ? r.json() : null)
    .then((data: { coaching_sentence?: string } | null) => {
      setCoachingSentence(data?.coaching_sentence ?? null)
    })
    .catch(() => {})
}, [port])
```

### FocusCard component

Add optional `coachingSentence` prop. Render it as small italic text at the bottom of the card if present:

```tsx
function FocusCard({ focus, champion, coachingSentence }: {
  focus: Focus
  champion: string
  coachingSentence?: string | null
}) {
  // existing render...
  {coachingSentence && (
    <p className="text-gray-400 text-[8px] italic mt-1.5 leading-relaxed">
      "{coachingSentence}"
    </p>
  )}
}
```

### Render call

Pass `coachingSentence` to `FocusCard`:

```tsx
{champ_data?.focus && !champ_data.no_history && (
  <FocusCard
    focus={champ_data.focus}
    champion={locked_champion}
    coachingSentence={coachingSentence}
  />
)}
```

## Edge Cases

- No focus card (not enough games): `/focus` returns null â†’ `coachingSentence` is null â†’ existing card unchanged
- Fetch fails (sidecar offline): catch sets nothing â†’ `coachingSentence` stays null â†’ existing card unchanged
- Empty coaching sentence: `data?.coaching_sentence ?? null` guards against empty string being shown
