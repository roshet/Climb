# Dynamic Counterfactual Generation — Design Spec
**Date:** 2026-04-19
**Status:** Approved

---

## Overview

Improve the per-game coaching notes generated by Gemini in two ways:

1. **(A) Pattern-aware coaching** — inject the player's cross-game patterns (recurring issues and win conditions) into the prompt so coaching notes can reference multi-game context ("you've missed objectives in 9 of your last 20 games, and here at 7:37 specifically...")
2. **(B) Better prompt structure** — add role-specific coaching instructions and a tighter per-note format (what happened → why it mattered → one concrete alternative action)

---

## Architecture

No new files. No schema changes. No new endpoints.

### Modified Files
- `sidecar/claude_client.py` — add `patterns` parameter to `generate_coaching_notes`, inject player context block into prompt, improve role-specific instructions
- `sidecar/backfill.py` — call `detect_patterns(db)` before `generate_coaching_notes` and pass result in
- `sidecar/tests/test_claude_client.py` — 3 new tests for pattern injection behavior

---

## Change 1: `generate_coaching_notes` signature

Current:
```python
def generate_coaching_notes(self, moments: list, game_context: dict, timeline: dict) -> list:
```

New:
```python
def generate_coaching_notes(
    self,
    moments: list,
    game_context: dict,
    timeline: dict,
    patterns: list | None = None,
) -> list:
```

`patterns` is a list of `PatternResult` objects from `pattern_detector.py`. It is optional — passing `None` (or omitting it) produces the same behavior as today. The parameter is added at the end with a default, so all existing call sites continue to work without modification.

---

## Change 2: Player context block in prompt (A)

When `patterns` is provided and non-empty, prepend a player context block to the Gemini prompt:

```
Player's cross-game patterns (last 20 games):
Recurring issues: objective_missed (9/20 games, 44% WR), tower_lost (8/20, 38% WR), plates_lost (5/20, 20% WR)
Win conditions: baron_secured (3/20, 100% WR)

When writing coaching notes, reference these patterns where relevant. For example, if the player has a recurring 'objective_missed' issue and this moment involves an objective, note that this is part of a broader pattern across their games.
```

Format rules:
- Recurring issues line: `"Recurring issues: "` followed by comma-separated `"{moment_type} ({games_seen}/{total_games} games, {int(win_rate_with*100)}% WR)"` entries
- Win conditions line: same format
- If only issues exist, omit the win conditions line (and vice versa)
- If `patterns` is `None` or empty, omit the entire player context block

## Change 3: Improved prompt instructions (B)

Current prompt rules block:
```
For each moment below, write a 3-4 sentence coaching note. Rules:
- Be specific to the {role.lower()} role
- Reference what was happening in the surrounding context
- Give one concrete, achievable alternative action
- Use encouraging language for positive moments (...)
- Describe game state for mistakes — don't moralize
- Keep each note to 3-4 sentences maximum
```

Replacement:
```
For each moment below, write a coaching note in exactly 3 sentences:
- Sentence 1: What happened and the game state at that moment (reference surrounding context)
- Sentence 2: Why it mattered — the impact on gold, objectives, or map control
- Sentence 3: One concrete, achievable alternative action specific to a {role.lower()} player

Role-specific guidance for {role.lower()}:
{_role_guidance(role)}

Tone: encouraging for positive moments (solo_kill, objective_secured, roam_kill, roam_assist, ward_kill, baron_secured, dragon_stack, gank_assist). Describe game state without moralizing for mistakes.
```

### Role guidance strings

```python
ROLE_GUIDANCE = {
    "TOP": "Focus on wave management, split push decisions, and teleport usage. Reference lane opponent and proximity of enemy jungler.",
    "JUNGLE": "Focus on pathing efficiency, objective timing, and gank setup. Reference which lanes were ahead/behind and dragon/baron spawn timers.",
    "MIDDLE": "Focus on roam timing, wave control before leaving lane, and mid-game priority. Reference lane opponent and river vision.",
    "BOTTOM": "Focus on CS efficiency, lane partner synergy, and positioning relative to support. Reference lane opponent and tower pressure.",
    "UTILITY": "Focus on vision control, peel timing, and roam opportunities. Reference support item usage and ward placements.",
}
```

Unknown roles fall back to a generic instruction: `"Focus on positioning, objective control, and decision-making at the moment of the event."`

## Change 4: `analyze_and_save_match` in `backfill.py`

Before calling `generate_coaching_notes`, call `detect_patterns`:

```python
try:
    game_patterns = detect_patterns(db_session)
except Exception:
    game_patterns = None

enriched = claude_client.generate_coaching_notes(moments, game_context, timeline_data, patterns=game_patterns)
```

The `try/except` ensures a pattern detection failure never blocks coaching note generation. If fewer than 3 games are in the DB (pattern detection returns `[]`), `[]` is passed as patterns — the prompt omits the context block since the list is empty.

## Error Handling

| Scenario | Behavior |
|---|---|
| `detect_patterns` raises in `backfill.py` | Caught, `game_patterns = None`, coaching proceeds without context |
| `patterns` is `[]` (fewer than 3 games) | Player context block omitted from prompt |
| Gemini call fails | Existing fallback to `counterfactual.enrich_moments` — unchanged |
| Unknown role in `ROLE_GUIDANCE` | Falls back to generic instruction string |

## Testing

`sidecar/tests/test_claude_client.py` — 3 new tests:

**`test_generate_coaching_notes_without_patterns`**
Call `generate_coaching_notes` with `patterns=None`. Assert the prompt sent to Gemini does NOT contain `"Player's cross-game patterns"`.

**`test_generate_coaching_notes_with_patterns`**
Call with a list of two `PatternResult` objects (one recurring_issue, one win_condition). Assert the prompt contains `"Recurring issues:"` and `"Win conditions:"` with the correct moment_type and game counts.

**`test_generate_coaching_notes_with_empty_patterns`**
Call with `patterns=[]`. Assert the prompt does NOT contain `"Player's cross-game patterns"`.

All three tests mock the Gemini client so no real API calls are made. They verify prompt content via `mock_client.models.generate_content.call_args`.

---

## Out of Scope

- Re-generating counterfactuals for already-stored games — stored moments are not updated retroactively
- Per-champion coaching guidance — role-level guidance is sufficient for this iteration
- Streaming responses from Gemini — existing batch approach is unchanged
